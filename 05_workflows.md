# 작업 워크플로우 (A / B / C)

작품마다 다른 모드. 셋 다 정당하지만 voice 보존 정도와 속도가 다르다.

---

## A. 사용자 직접, AI는 비계만

### 역할 분담
- **사용자**: 모든 문장 직접 작성. 1~14층 전부.
- **AI**: 자료조사, 구조 점검, 빠진 자리 짚기, 약점 진단. *문장 안 씀.*

### voice 보존
- 100% (사용자 손에서만 나옴)

### 속도
- 한 편당 몇 주~몇 달

### 어디 쓰나
- **시그니처 작품**
- **손 녹화 작품** (credentialing)
- 채널의 *기둥*. 1년에 1~2편.

### 절차
1. 사용자가 소재·주제·전제 결정
2. AI에게 자료조사·인접 사례 요청
3. 사용자가 비트 시트 작성 → AI가 약점 진단
4. 사용자가 처음부터 끝까지 직접 씀
5. AI가 안티 템플릿 표시 (재작성 안 함)
6. 사용자가 직접 수정

---

## B. 사용자 초고, AI는 선택적 폴리시

### 역할 분담
- **사용자**: 거친 초고 직접 작성. 모든 문장 자기 손.
- **AI**: 라인 단위 *지정된 작업만* — caboose 표시, *마치~처럼* 표시, exposition dump 자리 표시. *재작성 안 함.* 사용자가 채택 여부 결정.

### voice 보존
- 80% (사용자 문장이 살아남되 표면 정리만)

### 속도
- 한 편당 1~2주

### 어디 쓰나
- **채널 본편**. 정기 콘텐츠. 월 1~2편.
- 시그니처는 아니지만 사용자 작품이 분명한 것.

### 절차
1. 사용자가 KNOW·SHAPE 단계 완료
2. 사용자가 거친 초고 직접 작성 (voice 들어감)
3. AI가 안티 템플릿 적용 — *"이 자리 caboose"*, *"이 자리 평균값"*
4. 사용자가 표시 자리 직접 수정
5. AI가 한 번 더 점검
6. 사용자가 최종 결정

---

## C. AI 초고, 사용자가 voice 주입

### 역할 분담
- **AI**: 비트 시트 받아 거친 초고 작성
- **사용자**: 결정적 문장 5~7개, 디테일 3~5개 직접 박음. 나머지는 평균값 craft로 남음.

### voice 보존
- 30~40% (씨앗 정도)

### 속도
- 한 편당 몇 시간~며칠

### 어디 쓰나
- **부수 콘텐츠**. 채널 주변부.
- 짧은 영상, 소셜, 프로모.
- *작품성*보다 *양*이 우선되는 자리.

### 절차
1. 사용자가 비트 시트 작성
2. AI가 비트 시트 받아 거친 초고
3. 사용자가 *첫·마지막·결정적 문장 5~7개* 직접 작성
4. *디테일 3~5개* 직접 박음 (사용자만 가진 구체)
5. 나머지 라인은 평균값으로 남음

---

## 혼합 사용 권장

**한 가지 모드만 쓰면 위험**:
- A만 → 1년에 2편. 채널 모멘텀 부족.
- C만 → 평균값 양산. 사용자 voice 안 보임.
- B만 → 중간. 차별화 부족.

**현실적 분배**:

| 모드 | 빈도 | 용도 |
|---|---|---|
| A | 1~2편/년 | 시그니처. 손녹화. credentialing. |
| B | 월 1~2편 | 채널 본편. 주력 콘텐츠. |
| C | 필요할 때 | 보조·프로모·짧은 영상 |

A로 채널의 *기둥*을 세우고, B로 *몸*을 만들고, C로 *주변*을 채움.

---

## 핵심 원칙: 결정 자리는 분배 안 함

어떤 모드든 — **무게 자리는 사용자가**:
- 작품의 첫 문단·첫 문장
- 마지막 문장
- 작품의 핵심을 결정하는 5~7개 문장
- 시청자에게 *진짜로 하고 싶은 말*

이건 A·B·C 모두 동일. AI가 손대면 안 되는 자리.
