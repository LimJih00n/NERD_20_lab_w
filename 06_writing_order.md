# 작업 순서: KNOW → SHAPE → WRITE → 퇴고

문장부터 시작하지 않는다. 세 단계가 순차적으로 깊어진 후에야 문장이 시작된다.

---

## 1단계 — 알기 (KNOW)
*무엇을, 왜, 어떤 톤으로*. 문장 0개.

### 항목

- **소재** — 이 이야기로 무엇을 다룰 건가
- **왜 지금 / 왜 나** — 사용자가 *진짜로* 이 이야기를 해야 할 이유 (한 문장)
- **전제** — 한 문장 시놉시스 (사건 + 반전)
- **주제** — 작품이 *말하려는 것* (행위적 표현, 추상명사 금지)
- **레퍼런스** — 톤·구조·문제의식이 비슷한 작품 3~5개

### 분업

- **AI**: 자료조사, 인접 사례, 레퍼런스 후보 검색
- **사용자**: *왜 이 이야기를 내가* 의 결정. AI 못 함.

### 시간

- 단편: 30분 ~ 2시간
- 본격 작품: 며칠

---

## 2단계 — 모양 잡기 (SHAPE)
구조와 비트. 아직 문장 0개.

### 항목

- **인물 배치** — 누가 있고, 누가 누구와 어떤 관계
- **세계 설정** — 시대·장소·규칙 (최소만)
- **큰 골격** — 3막? 5장 우화? 비네트? 한 줄로 결정
- **비트 시트** — 5~7개 핵심 사건 (단편), 더 많이 (장편)
- **각 비트의 변화** — 이 비트에서 *무엇이 바뀌는가*
- **적용할 매크로 5~7개** — 어느 층에 어떤 매크로 호출할지

### 분업

- **AI**: 사용자 비트 시트의 *약한 인과·빠진 자리·반대 시점* 짚기. 매크로 후보 제시.
- **사용자**: 비트의 *순서·강도·생략* 결정. 매크로 *선택* 결정.

### 시간

- 단편: 1~2시간
- 본격 작품: 며칠 ~ 1주

---

## 3단계 — 쓰기 (WRITE)
드디어 문장. 다만 *처음부터 순서대로*가 아님.

### 권장 순서

1. **결정적 문장 먼저** — 작품의 마지막 한 줄, 또는 첫 문장. *진짜로 하고 싶은 한 마디* 를 먼저 박음. 이게 작품의 나침반.

2. **각 비트의 마지막 컷** — 비트가 어떻게 닫히는지 먼저. 닫힘이 결정되면 비트 안의 나머지가 거기로 향함.

3. **각 비트의 첫 컷** — 어떻게 여는지.

4. **나머지 채우기** — 비계 사이의 연결 조직.

### 분업 (모드에 따라)

- **A 모드**: 사용자가 1~4 모두 직접
- **B 모드**: 사용자가 1~4 직접, AI가 사후 표시만
- **C 모드**: 사용자가 1·2 직접 (결정적 문장과 비트 닫힘), AI가 3·4 (연결)

### 시간

- 단편: 며칠 (B 모드 기준)

---

## 4단계 — 퇴고

차수별로 다른 자리에 집중. 한 번에 모든 걸 고치려 하지 않는다.

### 1차: 구조

- 비트가 작동하나
- 인과가 연결되나
- 빠진 비트 / 잉여 비트
- 변화가 충분한가

### 2차: 정서

- 각 장면의 톤이 맞나
- 매크로가 의도대로 적용됐나
- 직격·암각 분배가 맞나

### 3차: 문장

- 안티 템플릿 적용 (`03_anti_templates.md`)
- caboose 제거
- 비유 안전망 제거
- 형용사 의존 줄이기

### 4차: 소리

- 읽어보면서 *입이 멈추는 자리* 표시
- 호흡이 끊기는 자리 표시
- 같은 단어 반복 표시

각 차수별 별도 세션. 같이 하지 않음.

---

## 함정 두 가지

### 함정 1: 1·2단계 건너뛰고 바로 쓰기

문장은 나오는데 *어디 가는지 모름*. 평균값 쪽으로 흘러감. *비트 시트가 없는 글은 exposition dump가 된다.*

### 함정 2: 1·2단계에 너무 오래 머묾

*쓰기를 미루는 방어*. 완벽한 outline 만들다 쓸 마음이 사라짐. 1·2단계는 *최소한*만, 3단계로 빨리 넘어가야 함.

**경계선**: 비트 시트의 *변화* 항목이 모든 비트에 한 문장씩 채워졌으면 — 3단계로 넘어가도 충분.

---

## 작업 시간 가이드 (5분 단편 기준)

| 단계 | 시간 | 누구 |
|---|---|---|
| 1. KNOW | 30분~2시간 | 사용자 + AI 자료조사 |
| 2. SHAPE | 1~2시간 | 사용자 + AI 비트 점검 |
| 3. WRITE | 3~5시간 (A모드 기준) | 사용자 |
| 4. 퇴고 1차 (구조) | 1시간 | 사용자 |
| 4. 퇴고 2차 (정서) | 1시간 | 사용자 |
| 4. 퇴고 3차 (문장) | 1~2시간 | 사용자 + AI 표시 |
| 4. 퇴고 4차 (소리) | 30분 | 사용자 |
| **합계** | **8~13시간** | |

B 모드면 3단계가 1~2시간으로 줄어 *6~10시간*. 채널 본편 1편 만드는 데 이 정도 들임.
