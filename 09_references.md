# 레퍼런스 라이브러리 — 동적 누적 로그

매크로의 [재료] 섹션을 채우고 시스템을 진화시키기 위한 *분해된 작품들의 살아 있는 로그*.

**기존(정적)** → **새로움(동적)**:

| 차원 | 정적 (이전) | 동적 (현재) |
|---|---|---|
| 형태 | 미리 채워둔 카탈로그 | *살아 있는 누적 로그* |
| 작동 | 사용자가 적은 작품에서만 고름 | 분해할 때마다 *AI가 검색·인접 작품 발견* |
| 확장 | 사용자 시간 안 들면 안 채워짐 | 분해 진행과 함께 자동 두꺼워짐 |
| 저장 | 한 파일 안에 다 | `references/<work>.md` 파일별 분해 |
| 깊이 | 작품명·한 줄 | *각 작품의 15층(0+14) 분해* + 검색 보강 |

= **분해 1회 = 시스템 1회 진화**. 자세히: `11_agent_system.md`.

## 동적 검색 — AI가 분해할 때 자동 보강

분해 진행 중 AI가 자동 검색:

1. **작품 자체 분석 글·리뷰** — 어떤 자리가 시장에서 받아들여졌나
2. **감독·작가 인터뷰** — 왜 이렇게 만들었는지의 의도
3. **흥행·평가 데이터** — 시장 차원 보강 (0층 시장 분해용)
4. **인접 작품** — 비슷한 hook·구조의 작품들
5. **학술적 분해** (필요 시) — 깊은 분석 추출

→ 09_references.md에 인덱스만 누적. 세부는 `references/<work>.md`.

## 분해 방법 — 15층 통과 (0층 + 14층)

```
[0층] 어떻게 셋(재미·시장·색채)을 동시에 잡았나
       - 재미 hook은 뭐고
       - 시장은 왜 따라왔고
       - 색채는 어느 자리에 박혔나
       - 셋 충돌 시 어디 무게 뒀나 (= trade-off 룰)
       ↓
[1·2] 전제·주제 추출
       ↓
[3·4·5] 세계·인물·골격
       ↓
[6·7·8] 시퀀스·신·비트 (장편이면 *대표 신 1~2개*만 깊게)
       ↓
[9·10·11] 시점·컷·음향
       ↓
[12·13·14] 정서·문단·문장 (대표 신의 *디테일*)
       ↓
[저장]   references/<work>.md 작성
[갱신]   04_macros.md — 새 매크로 후보 추가, 기존 매크로 [재료] 한 단락
[갱신]   09_references.md — 이 인덱스에 한 줄
```

각 층은 `02_layers.md`의 위계 알고리즘 따라감. 작품 분해도 작품 생성도 *위→아래* 일관.

## 추출 시 5단계 분해 (보조 도구)

각 장면에 대해 (`04_macros.md` 깊은 매크로 7 차원과 별도로 — 빠른 분해용):

1. **어떤 장면?** (3~5문장 묘사)
2. **시간 구조** (지속·끊김·실시간·시간 압축)
3. **무엇이 안 보였나** (작가가 일부러 뺀 것 — 보통 가장 중요)
4. **주된 감각** (시각? 청각? 시간? 어떤 감각이 과부하?)
5. **결과적 잔여** (작품이 그 사건에 대해 어떻게 다뤘나 — 즉시 잊음 vs 오래 머묾)

→ 5분해는 *후보 발견*용. *깎음*은 매크로의 7 차원으로 (`04_macros.md`).
