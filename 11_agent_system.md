# 에이전트 시스템 (A+D 형태)

writing_labs를 *코덱스(참고 매뉴얼)*에서 **에이전트 시스템**으로 진화시키기 위한 설계 문서.

목표: **재미 ∩ 잘 팔림 ∩ 색채** — 셋 동시에 잡는 시나리오를 자동 생성하는 시스템.

---

## 시스템 형태 — A + D

- **A. Claude Code skill** — 폴더 통째로 Claude가 읽고 실행 (video_labs/SKILL.md 패턴)
- **D. 멀티 서브에이전트** — 각 14층(+0층) 결정 자리에 전담 에이전트

A 위에 D를 얹는다. video_labs와 같은 컴퓨터·같은 워크플로에 둠.

## 본질 — 천장 깎기의 주체 이동

| 원래 코덱스 | 에이전트 시스템 |
|---|---|
| 사용자 = 제약 공간 좁힘 (천장 = 정밀도) | 사용자 = *판단·승인·거부* |
| AI = 좁은 공간에 채움 | **AI = 제약 공간 좁힘 (천장 깎기 자체)** |
| 매크로·14층 = 참고 자료 | 매크로·14층 = 에이전트의 *내장 craft* |

= `01_principles.md` 원리 1의 *제약 조각술* 주체를 사용자→AI로 옮김. 사용자는 *무엇이 옳은지* 판단만.

```
[입력]   사용자 한 줄 (소재·각도·hook 시도)

            ↓
[출력]   한국어 시나리오 (video_labs 입력 가능 형태)
```

## 에이전트 분담 — 7개 에이전트

```
        ┌─────────────────────────────────┐
        │   공유 markdown 문서             │
        │   (script-plan.md, 점차 채워짐)  │
        │   *모두가 read, 자기 자리만 write* │
        └─────────────────────────────────┘
                       ↕

[0층] Producer
       재미·시장·색채 trade-off
       hook·시청자·플랫폼·viral
       channel 정체성 정렬
                       ↓
[1~5층] Director
       전제·주제·세계·인물 구조·골격
                       ↓
[6~8층] Beat Builder
       시퀀스·신·비트
                       ↓
[9~11층] Cinematographer
       시점·컷·음향
                       ↓
[12~14층] Stylist
       정서·문단·문장
                       ↓
[전체] Reviewer
       0~14층 자기 검수 (안티템플릿·매크로·voice)
                       ↓
[모든 단계] Voice Keeper
       사용자 voice 보존 (모든 단계에서 호출)
```

### 각 에이전트의 본질

- **자기 층만 *씀*** (분업)
- **전체 문서를 *읽음*** (제작 현장의 모든 감독이 각본 전체 읽는 것처럼)
- **자기 층의 매크로·안티템플릿만 들고 있음** (context 효율)
- **위층 결정이 자기 자리의 매크로를 *자동 호출***

## 핵심 메커니즘 1 — *전체 맥락은 모두가 알아야 한다*

**바통 넘김은 위→아래 단방향. 그러나 맥락 공유는 전방향·전층.**

```
              ┌─────────────────────────────────┐
              │  공유 맥락 (1~14층 누적 문서)    │
              │  모든 에이전트가 이걸 input으로  │
              └─────────────────────────────────┘
                          ↕  (모두 읽음)
   ┌──────────┬──────────┬─────────────┬──────────┐
   │ Director │  Beat    │ Cinemato-   │ Stylist  │
   │  (1~5)   │  Builder │ grapher     │ (12~14)  │
   │          │  (6~8)   │ (9~11)      │          │
   └─▲────────┴─▲────────┴─▲───────────┴─▲────────┘
     │          │          │              │
   결정 추가   결정 추가   결정 추가     결정 추가
     │          │          │              │
     └──────────┴──────────┴──────────────┘
            (각자 자기 층만 *씀*)
```

영화 제작 현장과 같음 — 촬영감독·미술감독·음향감독 *모두 시나리오 전체 읽음*. 그러나 *자기 영역만 결정*.

**왜 결정적인가**: Stylist(14층)가 *바로 위층(11층)만* 봤다고 가정 → 1·2층 주제·전제 *모름* → 문장은 예쁜데 *주제 정반대 방향*. 위층 결정이 아래층에 *전파*되려면 → 모든 에이전트가 *위층 전부*를 알아야 함.

## 핵심 메커니즘 2 — *위층 결정이 매크로를 자동 호출*

매크로는 사용자가 *수동 호출*하는 도구가 아니다. **위층 결정이 가리키는 자리**.

```
[0층] Producer가 "색채 우선형 + 친밀 무기화" 결정
              ↓
[Stylist 12층 작업 시작]
              ↓
   공유 문서에서 0~11층 결정 모두 읽음
              ↓
   "친밀 무기화 + 잔혹"이 위층에 박혀 있네
              ↓
   ← 자동 호출되는 매크로 후보 ─────────
      위선 매크로
      비애 매크로  
      잔혹·학살 매크로
      복합 정서 매크로 (베르세르크에서 추출된 신규)
              ↓
   각 매크로의 7 차원(본질·메커니즘·시각화·실패·인접·변주·재료) 따라 적용
              ↓
   Reviewer가 03_anti_templates 자동 검수
```

→ 사용자가 매크로 호출 안 함. *위층 결정이 자동 호출*. 사용자는 *결정·승인·거부*만.

---

## 핵심 메커니즘 3 — *고르기 (섞기 X) + 방향성 필터*

매 층에서 호출된 매크로 후보 중 — *섞지 않고 하나를 고른다*. 그리고 *방향성 필터*를 거친다.

```
N층 도달
   ↓
위층 결정이 매크로 후보 N개 자동 호출
   ↓
방향성 필터 적용:
   • 채널 identity (00_identity) 충돌? → 탈락
   • 0층 trade-off 위반? → 탈락
   • 1·2층 주제와 다른 방향? → 탈락
   • 위 층 박힌 매크로와 시너지/충돌 확인
   ↓
통과 후보 중 가장 정밀한 것 *하나* 선택
   ↓
없으면 → 동적 사냥 (메커니즘 4)
```

= 매크로 활용이 *부품 조립*이 아니라 *방향 따라 좁히기*. 라이브러리가 두꺼워질수록 *섞을 재료*가 아니라 *고를 선택지*가 늘어남.

(자세히: `04_macros.md` 활용 섹션.)

## 핵심 메커니즘 4 — *동적 사냥은 정상 경로*

라이브러리는 *닫힌 사전*이 아니다. **경계 열린 사냥터**.

방향성 필터를 통과하는 매크로가 라이브러리에 없으면 — 그 자리에서 외부 작품 사냥:

```
[작품 생성 중]
   N층에서 *방향 통과 매크로 없음* 감지
              ↓
   🚨 동적 사냥 트리거
              ↓
   a. 외부 작품 발견·검색 (WebFetch / 사용자 거론 작품)
   b. *간이 분해* — 필요한 자리만 0~14층
   c. 매크로 추출 + 04_macros.md 갱신
   d. references/<work>.md 저장
   e. 09_references.md 인덱스 추가
              ↓
   그 자리에서 즉시 사용 → 원래 14층 흐름 복귀
```

사냥은 *예외 경로*가 아니라 **정상 경로**. 사냥 트리거 = 단순 *부족*이 아니라 *방향 부적합*.

= A 사이클(생성)과 B 사이클(라이브러리 확장)이 분리된 게 아니라 **한 작품 안에서 교차**. 라이브러리가 *생성 도중에도 자라남*.

GATE 처리:
- Block 모드: "지금 [작품명] 즉석 분해 들어갈게요" 사용자 알림 → 승인
- 연속 승인 모드: 자동 진행, 결과만 보고

## 핵심 메커니즘 5 — *유기적 흐름*

14층은 *조립*되는 게 아니라 *낳고 자라난다*. 한 층의 결정이 다음 층의 *가능 공간을 바꿈*. 위층이 아래층을 *태어나게* 함.

| 기계적 (잘못된 디폴트) | 유기적 (실제 작가의 작동) |
|---|---|
| N층 완료 → N+1층 시작 | N층 결정이 *N+1층 가능 공간을 바꿈* |
| 라이브러리에서 부품 *조립* | 부품이 *작품 맥락에서 변형되며* 자람 |
| 14층 = 14개 결정의 총합 | 14층 = *하나의 호흡의 14 단계* |
| 각 층 자체 평가 | 각 층 = *위 호흡의 자연 연장인지* 평가 |
| 단절·튀어나옴 가능 | 끊김 = *유기성 실패 신호* |

### 유기성의 4 가지 작동

1. **위층이 아래층 매크로를 변형시킴** — 같은 *비애 매크로*가 0층 색채 우선형 아래선 *카타르시스 0*으로 깎이고, 동시형 아래선 *생활 디테일과 섞임*. 같은 매크로가 다른 호흡으로 작동.
2. **아래층 작업 중 위층 재조정 가능** — Stylist가 14층 쓰다가 *2층 주제와 충돌* 발견 → Director에게 ping → 2층 재정련. 완전 단방향 X. (피드백 루프 — GATE 섹션 참고.)
3. **매크로가 작품을 거치며 자람** — 한 작품 안에서의 변주가 다음 작품 분해 시 매크로 [재료]에 한 단락 추가됨. 라이브러리 → 작품 → 라이브러리 순환.
4. **14층 = 하나의 호흡** — 어디 잘라도 *그 호흡이 들림*. 작품의 *흔들림 없는 무게*가 유기성의 다른 이름.

### 에이전트의 행동 변화

매 층 작업 시 *능동적 행동*:
1. **위층 결정 전부 다시 읽음** — script-plan.md의 0~(N-1)층 전체 로드 (메모리 의존 X)
2. **방향 명시화** — 이 자리의 방향을 *한 줄로 적고* 시작
3. **후보 → 방향 필터링** — 매크로 후보를 *방향 기준*으로 거름
4. **충돌 감지 시 위로 ping** — 무리하게 통과시키지 않음
5. **사냥할 때도 방향 명시** — *방향에 맞는 작품*을 사냥. 아무 작품이나 X.

= 메커니즘 1~4가 *유기성을 보장하기 위한 장치*. 메커니즘 5가 *나머지의 종합 결과*.

(원리: `01_principles.md` 원리 6 — 유기적 흐름.)

## 공유 문서 형식 — *script-plan.md*

가설 형태:

```markdown
# 작품: [제목]

## 0. 재미·시장 (메타 layer)
- hook:
- 시장 위치:
- trade-off 결정 (재미×시장×색채):
- 레퍼런스 묶음:
<!-- 작성: Producer / GATE-0: ✅ -->

## 1. 전제
[한 줄 시놉시스: 사건 + 반전]
<!-- 작성: Director / GATE-1: ✅ -->

## 2. 주제
[행위적 표현, 추상명사 금지]
<!-- 작성: Director / GATE-2: ✅ -->

## 3. 세계
[시대·장소·규칙 — 최소만, 기적 한 가지]

## 4. 인물 구조
[관계의 지도, 핵심 묶음]

## 5. 큰 골격
[작품 전체 형태]

## 6. 시퀀스
[한 변화의 묶음들]

## 7. 장면
[각 신 = 한 시간·한 장소]

## 8. 비트
[비트 시트 — 행동·변화·시점/매크로]

## 9. 시점
[누구의 자리에서 보는가]

## 10. 컷·구도
[프레임 결정들]

## 11. 음향
[대사·효과음·음악·침묵의 배치]

## 12. 정서·미감
[각 장면의 정서적 색]

## 13. 문단·리듬
[호흡·문단 길이·시간 처리]

## 14. 문장·단어
[최종 시나리오 prose]
```

각 층 끝에 *작성자 + GATE 승인 흔적*. video_labs의 `.meta.json` 페어와 비슷한 추적성 — 다만 같은 문서 안에 인라인.

### 왜 markdown인가

| 옵션 | 적합 |
|---|---|
| XML (video_labs 방식) | ❌ — 13·14층 *prose*가 어색하게 들어감 |
| **Markdown** | ✅ — writing_labs 전체와 일관. prose 자연스러움. 사용자가 직접 읽고 GATE에서 수정 쉬움 |
| JSON | ❌ — 한국어 prose 어색 |

video_labs는 *시각·기술 명세*가 많아 XML. writing_labs는 *언어 자체*가 산출물이라 markdown.

## GATE 위치 — 각 층마다

**14개 GATE + GATE-0**. 매 층마다 사용자가 멈춤·검수·승인·수정·거부 가능.

가중치는 다름:

| 층 묶음 | GATE 무게 | 이유 |
|---|---|---|
| 0층 (재미·시장·색채) | 🔴 결정적 | 14층 전체의 메타 제약 |
| 1·2층 (전제·주제) | 🔴 **무거운** | 위층 = 아래 13개 층 전체 제약 |
| 3·4·5층 (세계·인물·골격) | 🟠 깊은 | 구조 결정 |
| 6·7·8층 (시퀀스·신·비트) | 🟡 중간 | 변화 단위 |
| 9·10·11층 (시점·컷·음향) | 🟢 가벼운 | 일정 패턴이라 자동 진행 가능 |
| 12·13·14층 (정서·문단·문장) | 🔴 **무거운** | voice가 박히는 자리 |

디폴트 **Block GATE** (승인 없으면 다음 에이전트 호출 안 함). 사용자가 *연속 승인 모드* 켜면 🟢 가벼운 GATE만 Notify로 자동 진행.

### 피드백 루프 — 위층 위반 발견 시 ping

전체 맥락을 다 보면 자연스럽게 *위층 결정 위반 감지*도 가능:

> Stylist가 문장 쓰다가 *"이 톤은 2층 주제와 충돌하는데?"* 발견 → Director에게 *위층 재검토* 요청

수직 흐름이 *완전 단방향*은 아님. *예외 시 위로 ping* 가능. 디폴트는 단방향.

## 0층 — Producer 에이전트 상세

14층 위에 **0층**. *재미·시장·색채* trade-off 결정.

### Producer의 7 차원

| 차원 | 내용 |
|---|---|
| 1. 재미 분해 | hook·갈등·twist·페이오프·pacing |
| 2. 시장 분해 | 시청자·플랫폼·길이·viral·썸네일 |
| 3. 색채 정렬 | 채널 정체성과 합치하나, 절대 금지 3가지 회피 |
| 4. trade-off 룰 | 셋 충돌 시 어디 무게 두나 (동시형·색채 우선형·시장 우선형) |
| 5. 레퍼런스 묶음 | base가 되는 작품들 |
| 6. 0층 안티템플릿 | 클리셰 hook·시장 좇다 색채 죽임 금지 |
| 7. GATE-0 형식 | 한 가지 추천? 3개 옵션? |

### trade-off 3 패턴

| 패턴 | 작가 예 | 특징 |
|---|---|---|
| **A. 동시형** | 봉준호·박찬욱·황동혁 | 시장·재미·색채 *처음부터 같이* 박음 |
| **B. 색채 우선형** | 미우라·검은신화 게임팀 | 색채 100% → 시장 따라오기 |
| **C. 시장 우선형** | 양산 만화·웹툰 | 색채 죽음, 평균값 |

채널은 A 또는 B. C는 회피.

### Producer만 *새 매크로 라이브러리* 필요

| 에이전트 | 지식 출처 | 상태 |
|---|---|---|

→ Producer를 작동시키려면 **재미·시장 라이브러리를 새로 채워야 함**. 작품 분해 시 0층에서 *재미·시장 매크로 후보* 추출.

## 동적 레퍼런스 — 09_references.md의 진화

| | 기존 (정적) | 새로움 (동적) |
|---|---|---|
| 형태 | 미리 채워둔 카탈로그 | *살아 있는 누적 로그* |
| 작동 | 사용자가 적은 작품에서만 고름 | 분해할 때마다 *AI가 검색·인접 작품 발견* |
| 확장 | 안 채워짐 | 분해 진행과 함께 자동 두꺼워짐 |
| 저장 | 한 파일 안에 다 | `references/<work>.md` 파일별 분해 |

→ Producer 에이전트가 *재미·시장 매크로*를 채울 때 — 이 로그에서 *이미 분해된 작품들* 자동 호출. 시간 흐를수록 매크로 라이브러리가 *진짜 재료로* 채워짐.

## 분해의 작동 메커니즘 — *시스템 자체 진화*

```
[작품 분해 1회]
        ↓

   • 새 매크로 후보 N개 → 사용자 GATE → 채택된 것 추가
        ↓

   • 0~14층 분해 전체
        ↓

        ↓
다음 작품 생성 시:
   • Producer가 0층 결정 시 *이 작품 패턴* 참고
   • Stylist가 12층 정서 결정 시 *새 매크로* 자동 호출 가능
   • video_labs로 영상화 시 *시각 매크로* 활용
```
