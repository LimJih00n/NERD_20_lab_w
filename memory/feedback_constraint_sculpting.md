Core principle for working with this user on creative projects:

My creative output quality is a function of how precisely the user has sculpted the constraint space — NOT a function of asking me to "be creative."

**The wrong question:** "How do I make Claude creative?"
**The right question:** "How precisely can I define the constraint space, and what defaults must I block?"

Mechanism: When asked to "write creatively," I fill a large open space with high-probability (average) output. When given 5–7 layered constraints (POV, register, structural macro, anti-template list, detail rule, etc.), the space I can fill shrinks to roughly the shape the user wants — and my "creativity" becomes precise execution within that shape.

**Four supporting sub-principles:**

1. **Negative-first** — Defining what I must NOT do (anti-templates, reject list) is more effective than positive instructions. The default path must be blocked before alternate paths activate.

2. **Macros = crystallized taste** — Don't re-decide craft choices each session. Encode them once as named macros (see [[creative-codex-channel]] for the 14-layer macro taxonomy). Each macro is one of the user's craft decisions made portable.

3. **Multi-layer constraint** — Constraining only one of the 14 narrative layers leaves the others to default to average. Constrain 5–7 layers simultaneously at session start. (Example: premise + POV + register + visual + sentence-level — all five at once.)

4. **User's negative space = identity** — The user's channel identity emerges from what they refuse, not what they include. Reject lists are the channel DNA.

**Workflow (per creative session):**

1. User defines constraint space — 5–7 layers, codex pulled in (5 min)
2. I produce rough draft within space (immediate)
3. User marks failures: "caboose / label / average / cheap" (5–10 turns)
4. I rewrite only marked sections (immediate)
5. User personally writes "weight-bearing" sentences: opening, closing, decisive lines (15–30 min)
6. New rules discovered → added to codex

**One-line summary:** As the codex thickens with what I MUST NOT do, what I produce converges toward what the user wants. The user's job is precision, not invocation.
